# Руководство по встроенному браузеру в VS Code

## 1. Как запустить страницу во встроенном браузере

1. Поднимите локальный сервер для проекта.
   - Для простого статического сайта:
     - `py -m http.server 8000`
     - или `python -m http.server 8000`
   - Для React / Vite SPA:
     - `npm install`
     - `npm run dev`

2. Откройте страницу в встроенном браузере через URL.
   - Для статического файла: `http://127.0.0.1:8000/viewer.html`
   - Для Vite: обычно `http://127.0.0.1:5173/`

3. После этого можно работать с вкладкой через инструменты браузера:
   - открыть страницу;
   - кликать по элементам;
   - вводить текст;
   - переходить по ссылкам;
   - читать состояние страницы;
   - делать скриншоты.

---

## 2. Как управлять страницей из VS Code

Обычно используются следующие действия:

- `open_browser_page(url)`
  - открывает новую вкладку браузера;
  - полезно для проверки страницы после запуска сервера.

- `navigate_page(...)`
  - переход по URL, назад/вперёд, перезагрузка страницы.

- `click_element(...)`
  - кликает по нужному элементу;
  - можно указывать `selector` или `ref`.

- `type_in_page(...)`
  - вводит текст в поле;
  - подходит для форм, поиска, инпутов.

- `read_page(...)`
  - возвращает снимок текущего состояния страницы;
  - удобно для проверки, что UI открылся как ожидается.

- `screenshot_page(...)`
  - делает скриншот области или всей страницы.

---

## 3. Как это удобно использовать для React SPA

Для React SPA встроенный браузер особенно полезен, потому что можно:

1. Запустить dev-сервер:
   - `npm run dev`

2. Открыть страницу по адресу Vite:
   - `http://127.0.0.1:5173/`

3. Проверить UI вживую:
   - открыть меню;
   - нажать кнопки;
   - ввести данные в формы;
   - перейти по маршрутам;
   - проверить, что компонент перерисовывается после взаимодействия.

4. Использовать браузер как "живую проверку":
   - если UI не открывается, значит проблема в запуске сервера или в роутинге;
   - если кнопка не реагирует, можно проверить, что нужный компонент действительно доступен и не перекрыт overlay;
   - если страница падает, можно смотреть на ошибки в консоли и на снимок страницы.

---

## 4. Практический рабочий сценарий

1. Запустить сервер.
2. Открыть страницу во встроенном браузере.
3. Проверить, что загрузился основной экран.
4. Выполнить нужное действие:
   - клик по кнопке;
   - ввод в поле;
   - переход по маршруту;
   - проверка ответа/изменения UI.
5. Если что-то не работает, повторить с другой вкладкой или перезагрузить страницу.

---

## 5. Частые проблемы и решения

- Страница не открывается:
  - сервер не запущен;
  - адрес неправильный;
  - порт уже занят.

- Клик не срабатывает:
  - элемент перекрыт overlay;
  - нужно выбрать более точный `selector`.

- React SPA не переключает маршрут:
  - проверьте, что dev-сервер работает;
  - откройте правильный URL (включая route, если он не корневой).
